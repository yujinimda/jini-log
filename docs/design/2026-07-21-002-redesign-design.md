# 002 디자인·사용감 개편 설계

- 날짜: 2026-07-21
- 상태: 사용자 승인 완료 (brainstorming 산출물, 비주얼 컴패니언으로 톤 확정)
- 범위: 공개 블로그 + 어드민의 시각 디자인·사용감 개선. 001 MVP의 기능 위에 얹는 개편.

## 문제 정의

001 MVP는 기능적으로 완성됐지만 (사용자 피드백):
1. **톤 부재** — 특정 화면이 아니라 전반적으로 "디자인 안 한 기본값" 인상, 아이덴티티 없음
2. **사용감 부족** — 검색·태그 탐색·목차 같은 "당연한 편의"가 없고 가독성이 떨어짐 (기능 사용감의 기준: 우아한형제들 기술블로그)

## 핵심 결정

| 결정 | 선택 | 이유 |
| --- | --- | --- |
| 시각 톤 | **B1 순정 미디엄** — 세리프 제목 + 장식 제로 + 흑백 농도 | 비주얼 컴패니언 3안(터미널/미디엄/볼드 히어로) → B → B 세부 3안 중 B1 선택. 개성은 장식이 아니라 인터랙티브 콘텐츠가 만든다 |
| 구현 기반 | **shadcn/ui + Tailwind 유지** | 컴포넌트를 레포에 코드로 복사(의존·베타 리스크 없음), Command(⌘K)가 검색 요구에 직결, 기존 Tailwind 코드와 공존 |
| Astryx | **기각** (직전 결정 번복) | 목표가 "타이포 중심 톤 + 편의 기능"으로 구체화되자 StyleX 전면 재작성 비용 대비 이득 없음. 베타 + Next 16 호환 리스크 |
| 검색 방식 | 빌드 타임 정적 인덱스 + 클라이언트 필터 | 서버·DB 추가 없음, 수백 글 규모 충분. 001 스펙의 "사이트 내 검색 범위 밖"을 002로 편입하는 스펙 변경 |

## 디자인 파운데이션

- **색**: 순백 배경 + 잉크 블랙(#111) + 회색 스케일. 액센트 컬러 없음 — 링크·상태도 흑백 농도로 구분
- **타이포**: 제목 Noto Serif KR(세리프) / 본문 Pretendard. 둘 다 가변 폰트 **self-host** (구글 폰트 런타임 의존 제거, OG 이미지 폰트 통일 — 001의 네트워크 의존 폴백 문제도 해소)
- **본문 스케일**: 17px · 행간 1.8 · 본문 폭 42rem
- `app/(blog)/blog.css`의 `.prose`를 이 토큰으로 재정의 → 에디터 프리뷰에 자동 반영 (001의 공유 구조)
- 어드민은 산세리프 도구 톤 — 세리프는 콘텐츠(프리뷰) 영역에만

## 공개 블로그

- **헤더**: 좌 세리프 로고 "지니로그" / 우 [태그 · 검색(⌘K) · RSS] — 얇고 조용하게
- **홈**: 글 카드 = 세리프 제목 + 요약 2줄 + 날짜·**읽기시간(자동 계산)**·태그. 구분선 리듬, 박스·그림자 없음
- **태그**: `/tags` 인덱스 페이지 + 헤더 링크 (칩 클라우드 안 함 — B1의 조용함 유지)
- **글 상세**: 세리프 대제목·메타 → 본문 → **우측 고정 목차**(데스크톱, 스크롤 추적 하이라이트) / 모바일 상단 접이식 → 하단 **이전/다음 글**

## ⌘K 검색

- 빌드 타임에 발행 글 전체 → 정적 인덱스 JSON (slug·제목·설명·태그·본문 앞 500자)
- shadcn **Command 다이얼로그**: ⌘K 또는 헤더 버튼 → 즉시 필터 → 엔터 이동
- 매칭: 한글 친화적 단순 포함 검색 (fuzzy 라이브러리 없이 시작 — YAGNI)
- 검색 UI는 흑백 도구 톤 (세리프 없음)

## 어드민 사용감

- shadcn 부품 교체: `window.confirm()` → **Dialog** (발행취소·삭제·덮어쓰기), 텍스트 알림 → **Toast**, 대시보드 → **Table**(정렬), 버튼·인풋 통일
- 에디터: 상단 액션바 고정, 마지막 저장 시각 상시 표시, 프리뷰 패널 토글

## 검증·경계

- 기존 E2E 61케이스 = 회귀 안전망. 셀렉터 변화는 레인 D 테스트와 함께 수정
- 신규 E2E: ⌘K 검색 플로우, TOC 추적, Dialog 확인 플로우

- **범위 밖**: 다크모드, 댓글, 인기글 정렬, Astryx
