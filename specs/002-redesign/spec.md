**Feature Branch**: `002-redesign`

**Created**: 2026-07-21

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 독자가 원하는 글을 빠르게 찾는다 (Priority: P1)

독자는 어느 공개 화면에서든 단축키(⌘K / Ctrl+K) 또는 헤더의 검색 버튼으로 검색창을 열고, 입력하는 즉시 제목·요약·태그·본문이 매칭된 글 목록을 보고, 선택해서 글로 이동한다. 태그 인덱스 페이지에서는 전체 태그와 태그별 글 수를 한눈에 보고 탐색한다. 홈의 글 목록은 제목·요약·발행일·읽기시간·태그를 보여줘 클릭 전에 글을 가늠할 수 있다.

**Why this priority**: "편한 것들이 없다"는 피드백의 핵심. 콘텐츠가 쌓일수록 탐색 부재의 비용이 커진다.

**Independent Test**: 발행 글 여러 개 상태에서 검색으로 특정 글 도달, 태그 인덱스 탐색, 목록 카드 정보 확인만으로 검증 가능.

**Acceptance Scenarios**:

1. **Given** 공개 화면 어디서든, **When** ⌘K(또는 헤더 검색 버튼)를 누르면, **Then** 검색창이 열리고 입력 즉시 매칭 글 목록이 갱신된다.
2. **Given** 검색 결과에서, **When** 항목을 선택(엔터/클릭)하면, **Then** 해당 글로 이동한다.
3. **Given** 매칭되는 글이 없으면, **When** 검색어를 입력해도, **Then** "결과 없음" 상태가 명확히 표시된다.
4. **Given** 초안이 존재해도, **When** 검색하면, **Then** 발행 글만 결과에 나타난다.
5. **Given** 태그 인덱스 페이지에서, **When** 태그를 선택하면, **Then** 해당 태그의 글 목록으로 이동한다.
6. **Given** 홈 목록에서, **When** 글 카드를 보면, **Then** 제목·요약(최대 2줄)·발행일·읽기시간·태그가 표시된다.

---

### User Story 2 - 독자가 글을 편하게 읽는다 (Priority: P1)

글 상세는 세리프 제목과 넉넉한 본문 타이포(크기·행간·본문 폭)로 읽기에 최적화되고, 장식 없는 흑백 톤으로 콘텐츠가 주인공이 된다. 긴 글에서는 목차로 구조를 파악하고 원하는 절로 바로 이동하며, 현재 읽는 위치가 목차에 표시된다. 글 끝에서는 이전/다음 글로 자연스럽게 이어 읽는다.

**Why this priority**: "가독성이 떨어진다"는 피드백의 직접 대응. 블로그의 본질 가치.

**Independent Test**: 절이 여러 개인 발행 글 하나로 타이포·목차·이전/다음 모두 검증 가능.

**Acceptance Scenarios**:

1. **Given** 글 상세에서, **When** 화면을 보면, **Then** 세리프 제목·산세리프 본문·흑백 톤·넉넉한 행간과 제한된 본문 폭이 적용돼 있다.
2. **Given** 절 제목이 있는 긴 글에서, **When** 넓은 화면으로 보면, **Then** 목차가 본문 옆에 고정 표시되고 스크롤에 따라 현재 절이 하이라이트된다.
3. **Given** 같은 글을 좁은 화면으로 보면, **When** 목차를 열면, **Then** 접이식 목차가 상단에 제공된다.
4. **Given** 목차에서, **When** 절을 선택하면, **Then** 해당 절로 스크롤 이동한다.
5. **Given** 글 끝에서, **When** 하단을 보면, **Then** 이전/다음 글(발행일 순) 링크가 표시된다. 첫 글·마지막 글에서는 없는 방향이 생략된다.
6. **Given** 에디터 미리보기에서, **When** 글을 작성하면, **Then** 새 타이포 톤이 발행 모습과 동일하게 적용된다 (001 "프리뷰 = 발행" 원칙 계승).

### User Story 3 - 운영자가 어드민을 쾌적하게 쓴다 (Priority: P2)

파괴적 액션(삭제·발행취소·덮어쓰기)의 확인은 브라우저 기본 팝업이 아닌 앱 안의 다이얼로그로 이뤄지고, 액션 결과(성공·실패)는 화면 구석 토스트로 알려준다. 대시보드 글 목록은 제목·날짜·조회수 기준으로 정렬할 수 있다.

**Why this priority**: 운영자 1인의 반복 경험 개선 — 독자 가치(P1) 다음.

**Independent Test**: 대시보드에서 정렬·발행취소·삭제 플로우만으로 검증 가능.

**Acceptance Scenarios**:

1. **Given** 대시보드에서, **When** 삭제/발행취소를 누르면, **Then** 앱 내 확인 다이얼로그가 뜨고 취소/확인을 선택할 수 있다.
2. **Given** 어떤 어드민 액션이든, **When** 완료(성공/실패)되면, **Then** 결과가 토스트로 표시된다.
3. **Given** 대시보드 목록에서, **When** 컬럼(제목/발행일/조회수)을 선택하면, **Then** 해당 기준으로 정렬이 토글된다.

---

### Edge Cases

- 검색 결과 0건 → 명시적 "결과 없음" 표시 (빈 화면 금지).
- 절 제목이 없는 글 → 목차 영역 자체를 표시하지 않는다.
- 발행 글이 1개뿐 → 이전/다음 내비 양쪽 생략.
- 요약이 길어도 카드에서 최대 2줄로 잘리고 말줄임 처리된다.
- 검색·목차 기능이 실패해도 글 열람은 정상 동작해야 한다 (점진적 향상).
- 어드민 액션 실패 시 토스트에 실패 사유가 표시되고 화면 상태가 오염되지 않는다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 독자는 모든 공개 화면에서 단축키(⌘K/Ctrl+K) 또는 헤더 버튼으로 사이트 내 검색을 열 수 있어야 한다.
- **FR-002**: 검색은 발행 글의 제목·요약·태그·본문을 대상으로 하며, 입력 즉시 결과가 갱신되고, 선택 시 해당 글로 이동한다. 결과 없음 상태가 명시된다.
- **FR-003**: 초안은 어떤 경우에도 검색 결과에 노출되지 않아야 한다.
- **FR-004**: 검색은 별도 서버·저장소 추가 없이 동작해야 하며, 독자 페이지의 초기 로딩을 늦추지 않아야 한다.
- **FR-005**: 전체 태그와 태그별 글 수를 보여주는 태그 인덱스 페이지가 있고, 헤더에서 접근 가능해야 한다.
- **FR-006**: 글 목록 카드는 제목·요약(최대 2줄)·발행일·읽기시간·태그를 표시해야 한다.
- **FR-007**: 읽기시간은 본문 분량으로 자동 계산(분 단위)되어야 한다.
- **FR-008**: 글 상세는 절 제목 기반 목차를 제공한다 — 넓은 화면은 본문 옆 고정 + 현재 절 하이라이트, 좁은 화면은 접이식. 절 제목이 없으면 목차를 표시하지 않는다.
- **FR-009**: 글 상세 하단에 발행일 순 이전/다음 글 내비게이션을 제공한다.
- **FR-010**: 모든 공개 화면은 확정된 시각 톤을 일관 적용한다 — 세리프 제목, 산세리프 본문, 무채색(액센트 컬러 없음), 넉넉한 읽기 스케일. 상세 값은 설계 문서(docs/design/2026-07-21-002-redesign-design.md)가 기준.
- **FR-011**: 에디터 미리보기는 새 타이포 톤을 발행 렌더와 동일하게 반영해야 한다 (001 FR-002 계승).
- **FR-012**: 어드민의 파괴적 액션 확인은 브라우저 기본 팝업이 아닌 앱 내 다이얼로그여야 한다.
- **FR-013**: 어드민 액션의 성공·실패는 토스트로 통지되고, 실패 시 사유가 표시되어야 한다.
- **FR-014**: 대시보드 글 목록은 제목·발행일·조회수로 정렬 가능해야 한다.
- **FR-015**: 이 개편으로 001의 기능·검색엔진 노출·성능이 저하되지 않아야 한다.

### Key Entities

- **검색 항목(Search Entry)**: 발행 글에서 파생되는 탐색용 요약 정보 — 제목, 요약, 태그, 본문 발췌, 주소. 발행 시점 기준으로 갱신된다.
- **목차 항목(TOC Entry)**: 글 본문의 절 제목에서 파생 — 제목 텍스트, 계층, 위치.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 독자는 어떤 공개 화면에서든 5초 이내(검색 열기→입력→선택)에 원하는 글에 도달할 수 있다.
- **SC-002**: 검색 입력에 대한 결과 갱신이 체감 지연 없이(즉시) 일어난다.
- **SC-003**: 발행 글 100%의 목록 카드에 요약·발행일·읽기시간이 표시된다.
- **SC-004**: 개편 후에도 대표 성능·SEO 진단 점수 90점 이상이 유지된다 (001 SC-003 계승).
- **SC-005**: 001의 자동화 테스트 전체가 개편 후에도 통과한다 (기능 회귀 0).
- **SC-006**: 어드민 파괴적 액션의 확인 UI가 100% 앱 내 다이얼로그로 제공된다 (브라우저 기본 팝업 0건).

## Assumptions

- 시각 톤의 구체 값(폰트·크기·색 농도)은 설계 문서가 단일 기준이며, 스펙은 "일관 적용" 여부만 검증한다.
- 검색 품질은 "포함 매칭" 수준으로 시작한다 — 오타 교정·유사어는 범위 밖.
- 다크모드, 댓글, 인기글 정렬은 범위 밖 (001과 동일).
- 어드민의 시각 톤은 공개 블로그와 달리 도구적 명료함을 우선한다 (세리프는 콘텐츠 미리보기 영역에만).
- 001 스펙의 "사이트 내 검색 범위 밖" 가정은 이 스펙으로 대체된다.
- 무채색 원칙의 예외는 본문 안 콘텐츠 컴포넌트(Callout·인터랙티브 데모)뿐이다 — 페이지 크롬(헤더·카드·링크·내비)은 무채색, 컴포넌트는 절제된 색 허용. Callout은 톤다운 파스텔로 재조정 (grilling 확정).
- 읽기시간은 한글 기준 분당 500자로 계산하고 1분 미만은 "1분"으로 표시한다 (grilling 확정).
- 카드 요약은 frontmatter description을 사용한다 — 본문 자동 발췌 없음 (grilling 확정).
- "이전 글" = 더 먼저 발행된 글, "다음 글" = 더 나중에 발행된 글 (grilling 확정).
