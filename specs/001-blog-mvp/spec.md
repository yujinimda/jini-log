**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 운영자가 블로그에서 직접 글을 작성·발행한다 (Priority: P1)

운영자는 배포된 블로그의 관리 화면에서 마크다운으로 글을 쓴다. 화면 한쪽에서 실제 발행 모습과 동일한 미리보기를 보면서 작성하고, 제목·요약·태그는 별도 입력 필드로 채운다. "초안 저장"으로 보관하거나 "발행"으로 공개한다. 발행하면 수 분 내 공개 페이지에 글이 나타난다.

**Why this priority**: "블로그에서 직접 적을 수 있어야 한다"가 이 프로젝트의 1번 요구사항이며, 콘텐츠가 만들어져야 나머지 기능(열람·SEO·대시보드)이 의미를 가진다.

**Independent Test**: 관리 화면에서 새 글을 작성해 발행하고, 공개 페이지에서 해당 글이 노출되는지 확인하는 것만으로 완결적으로 검증 가능.

**Acceptance Scenarios**:

1. **Given** 인증된 운영자가 작성 화면에 있고, **When** 마크다운 본문과 제목·요약·태그를 입력하면, **Then** 미리보기가 실제 발행 모습과 동일하게 실시간 갱신된다.
2. **Given** 작성 중인 글이 있을 때, **When** "초안 저장"을 누르면, **Then** 글이 초안으로 보관되고 공개 페이지에는 노출되지 않는다.
3. **Given** 초안 또는 새 글이 있을 때, **When** "발행"을 누르면, **Then** 수 분 내 공개 페이지 목록과 상세에 글이 나타난다.
4. **Given** 형식 오류가 있는 본문(잘못된 컴포넌트 태그 등)일 때, **When** 저장 또는 발행을 시도하면, **Then** 저장이 거부되고 오류 위치가 표시되며 사이트는 계속 정상 동작한다.
5. **Given** 저장 요청이 실패했을 때, **When** 운영자가 화면을 새로고침해도, **Then** 작성 중이던 내용이 복원된다.
6. **Given** 작성 중인 글이 있을 때, **When** 이미지를 붙여넣거나 업로드하면, **Then** 이미지가 글과 함께 보관되고 본문에 자동 삽입되어 미리보기에 표시된다.
7. **Given** 이미 발행된 글을 수정할 때, **When** 저장하면, **Then** 즉시 재발행되어 수 분 내 공개본이 갱신된다.

---

### User Story 2 - 독자가 글을 읽고 인터랙티브 요소를 조작한다 (Priority: P1)

독자는 블로그 홈·목록·태그 페이지에서 글을 탐색하고 상세 페이지에서 읽는다. 글 안에는 코드 블록(복사 버튼 포함)과 저자가 심어둔 인터랙티브 데모가 있으며, 독자는 설치나 로그인 없이 브라우저에서 바로 조작하면서 내용을 이해한다.

**Why this priority**: 독자 열람은 블로그의 존재 이유이고, "만지면서 이해하는 글"이 이 블로그의 차별점.

**Independent Test**: 발행된 글(인터랙티브 요소 포함 샘플)을 공개 페이지에서 열어 요소를 조작해보는 것으로 독립 검증 가능.

**Acceptance Scenarios**:

1. **Given** 발행된 글이 있을 때, **When** 독자가 홈/목록/태그 페이지에 접근하면, **Then** 글 목록이 최신순으로 노출되고 각 글로 이동할 수 있다.
2. **Given** 인터랙티브 요소가 포함된 글일 때, **When** 독자가 해당 요소를 클릭·입력 등으로 조작하면, **Then** 요소가 즉시 반응하며 페이지 이동 없이 동작한다.
3. **Given** 코드 블록이 포함된 글일 때, **When** 복사 버튼을 누르면, **Then** 코드가 클립보드에 복사된다.

---

### User Story 3 - 검색엔진이 글을 수집·노출한다 (Priority: P2)

검색엔진 크롤러가 사이트를 방문하면 모든 발행 글의 제목·요약·대표 이미지·구조화된 메타데이터를 수집할 수 있고, 사이트 전체 색인 목록(사이트맵)과 구독 피드가 제공된다. 글을 SNS에 공유하면 제목 기반의 미리보기 이미지가 자동으로 표시된다.

**Why this priority**: "검색에 잘 걸리게"가 2번 요구사항. 다만 글이 존재해야 성립하므로 P1 다음.

**Independent Test**: 발행된 글의 페이지 소스에서 메타데이터·구조화 데이터를 확인하고, 사이트맵·피드 경로가 유효한 문서를 반환하는지 확인.

**Acceptance Scenarios**:

1. **Given** 발행된 글이 있을 때, **When** 크롤러가 글 페이지를 요청하면, **Then** 글별 제목·요약·공유용 메타데이터와 구조화 데이터(기사 유형)가 응답에 포함된다.
2. **Given** 사이트가 배포되어 있을 때, **When** 사이트맵·로봇 규칙·구독 피드 경로를 요청하면, **Then** 발행 글이 모두 반영된 유효한 문서가 반환된다.
3. **Given** 대표 이미지를 따로 만들지 않은 글일 때, **When** SNS에 링크를 공유하면, **Then** 글 제목이 들어간 미리보기 이미지가 자동 생성되어 표시된다.

---

### User Story 4 - 운영자가 대시보드에서 글과 조회수를 관리한다 (Priority: P3)

운영자는 대시보드에서 발행 글과 초안 목록, 글별 조회수와 기간별 추이를 본다. 초안 목록은 "글감 인박스" 역할을 하며, 외부 도구가 규약된 방식으로 초안을 넣으면 별도 작업 없이 목록에 나타난다.

**Why this priority**: 운영 편의 기능으로, 작성·열람·SEO가 갖춰진 뒤에 가치가 생긴다.

**Independent Test**: 발행 글을 여러 번 조회한 뒤 대시보드에서 글 목록·조회수·추이가 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 발행 글과 초안이 있을 때, **When** 운영자가 대시보드에 접근하면, **Then** 발행 글과 초안이 구분되어 목록으로 표시되고 글별 조회수와 최근 발행일이 보인다.
1-1. **Given** 발행된 글이 있을 때, **When** 운영자가 발행취소하면, **Then** 글이 초안으로 돌아가 공개 페이지에서 내려간다. 삭제를 선택하면 확인 절차 후 글이 제거되며, 변경 이력에는 보존된다.
2. **Given** 독자가 글을 조회했을 때, **When** 대시보드를 확인하면, **Then** 해당 글의 조회수가 누적되어 있고 기간별 추이를 볼 수 있다.
3. **Given** 운영자 본인이 로그인 상태로 글을 조회했을 때, **When** 대시보드를 확인하면, **Then** 그 조회는 조회수에 포함되지 않는다.
4. **Given** 외부 도구가 규약된 위치에 초안을 추가했을 때, **When** 대시보드를 확인하면, **Then** 해당 초안이 초안 목록에 나타난다.

---

### Edge Cases

- 형식 오류가 있는 본문을 저장·발행하려 하면 저장 전 검증에서 거부되고 오류 위치가 안내된다. 잘못된 콘텐츠가 사이트 전체를 깨뜨리는 일은 없어야 한다.
- 저장 요청이 외부 요인으로 실패해도 작성 중 내용은 유실되지 않는다(자동 임시 보관에서 복원).
- 같은 제목(슬러그)의 글이 이미 존재하면 덮어쓰기 여부를 확인받는다.
- 미인증 사용자가 관리 화면·관리 기능에 접근하면 차단된다. 허용된 운영자 계정은 정확히 1개.
- 조회수 기록이 실패해도 독자의 글 열람에는 영향이 없다.
- 발행 직후 반영까지 수 분의 지연이 있으며, 관리 화면은 반영 대기 상태임을 안내한다.
- 다른 경로(외부 도구·수동)로 콘텐츠가 변경된 경우에도 관리 화면은 항상 최신 콘텐츠를 기준으로 편집을 시작한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 독자는 홈·글 목록·태그별 목록·글 상세 페이지를 인증 없이 열람할 수 있어야 한다.
- **FR-002**: 운영자는 관리 화면에서 마크다운으로 본문을 작성하고, 실제 발행 모습과 동일한 실시간 미리보기를 볼 수 있어야 한다.
- **FR-003**: 제목·요약·태그 등 글 메타정보는 본문과 분리된 입력 필드로 받아야 하며, 메타정보 형식 오류로 발행물이 깨지지 않아야 한다.
- **FR-004**: 글은 초안과 발행 두 상태를 가지며, 초안은 공개 페이지에 노출되지 않아야 한다. 발행 시 수 분 내 공개 페이지에 반영되어야 한다.
- **FR-005**: 글에는 저자가 등록해둔 인터랙티브 요소를 포함할 수 있어야 하고, 해당 요소는 공개 페이지와 작성 미리보기 양쪽에서 동일하게 동작해야 한다.
- **FR-006**: 저장·발행 전에 본문 형식을 검증하고, 실패 시 저장을 거부하며 오류 위치를 표시해야 한다.
- **FR-007**: 작성 중 내용은 자동으로 임시 보관되어, 저장 실패나 화면 이탈 후에도 복원 가능해야 한다.
- **FR-008**: 관리 화면과 모든 관리 기능은 사전에 허용된 단일 운영자 계정으로만 접근 가능해야 한다.
- **FR-009**: 모든 콘텐츠 변경(초안 저장·발행·수정)은 변경 이력이 보존되어 누가·언제·무엇을 바꿨는지 추적 가능해야 한다.
- **FR-010**: 글 조회는 글별·일자별로 페이지로드당 1회 단순 누적으로 집계되어야 하며, 운영자 본인(로그인 세션)과 알려진 봇(크롤러)의 조회는 집계에서 제외되어야 한다. 방문자 개인정보는 저장하지 않는다.
- **FR-011**: 대시보드는 발행 글·초안 목록, 글별 조회수, 기간별 추이를 표시해야 한다.
- **FR-012**: 발행된 모든 글은 검색엔진용 메타데이터(제목·요약·공유 이미지·기사 구조화 데이터)를 제공해야 하고, 사이트맵·로봇 규칙·구독 피드가 자동 생성·갱신되어야 한다.
- **FR-013**: 공유용 대표 이미지는 글 제목을 기반으로 자동 생성되어야 한다(수동 제작 불필요).
- **FR-014**: 외부 도구가 규약된 위치·형식으로 초안을 추가하면, 추가 연동 작업 없이 대시보드 초안 목록에 나타나야 한다. 형식 오류가 있는 초안은 목록에 오류로 표시되되 사이트 동작에는 영향을 주지 않아야 한다.
- **FR-015**: 운영자는 에디터에서 이미지를 붙여넣기·업로드로 삽입할 수 있어야 하며, 이미지는 글과 같은 저장소에 보관되어 글과 함께 버전 관리되어야 한다.
- **FR-016**: 각 글은 영문 소문자·숫자·하이픈으로 구성된 고유 주소(slug)를 가지며, 운영자가 작성 시 직접 입력한다. 발행 후에는 변경할 수 없다.
- **FR-017**: 운영자는 발행된 글을 발행취소(초안으로 되돌리기)하거나 삭제할 수 있어야 한다. 발행취소된 글은 공개 페이지에서 즉시 제외 대상이 되고, 삭제는 확인 절차를 거치며 변경 이력에는 보존된다.
- **FR-018**: 발행된 글의 수정 저장은 즉시 재발행을 의미한다. 별도의 "수정 초안" 상태는 두지 않는다.

### Key Entities

- **글(Post)**: 블로그의 핵심 콘텐츠. 제목, 요약, 작성일, 태그, 본문(마크다운 + 인터랙티브 요소)을 가지며 상태는 초안 또는 발행.
- **인터랙티브 요소(Interactive Component)**: 저자가 사전에 등록한, 글 본문에 삽입 가능한 조작형 콘텐츠 단위. 등록은 개발 작업으로만 가능하고 글에서는 사용만 한다.
- **조회 기록(View Record)**: 글별·일자별 조회 수 누적치. 운영자 세션의 조회는 제외.
- **운영자(Operator)**: 관리 화면에 접근 가능한 유일한 사용자. 인증된 세션으로 식별.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 운영자가 관리 화면에서 새 글 작성을 시작해 발행 버튼을 누르기까지 별도 도구 없이 완료할 수 있고, 발행 후 5분 이내에 공개 페이지에서 글이 확인된다.
- **SC-002**: 독자는 설치·로그인 없이 브라우저만으로 글 내 인터랙티브 요소를 조작할 수 있고, 조작에 대한 반응이 즉시(체감 지연 없이) 나타난다.
- **SC-003**: 발행된 글 100%가 검색엔진 크롤러 기준 필수 메타데이터·구조화 데이터·사이트맵 항목을 갖추며, 대표 SEO 진단 도구에서 90점 이상을 받는다.
- **SC-004**: 형식 오류가 있는 본문의 저장 시도는 100% 저장 전 단계에서 거부되며, 잘못된 콘텐츠로 인해 공개 사이트가 중단되는 사고가 0건이다.
- **SC-005**: 운영자 본인의 조회는 조회수에 0건 반영된다(로그인 상태에서 반복 조회 후 수치 불변으로 검증).
- **SC-006**: 저장 실패·화면 이탈 상황에서 작성 중이던 내용의 유실이 0건이다.

## Assumptions

- 운영자는 1명이며, 마크다운 문법에 익숙한 개발자다.
- 발행 후 공개 반영까지 수 분(약 1~2분)의 지연은 수용한다 (브레인스토밍에서 확정).
- 인터랙티브 요소의 신규 제작은 개발 환경에서의 코드 작업으로만 이루어지며, 관리 화면에서는 기존 요소의 사용만 지원한다 (확정된 경계).
- 콘텐츠 변경 이력 보존은 버전 관리 저장소를 통해 충족한다 (설계 문서의 파일 + git 결정).
- 댓글, 사이트 내 검색, 인기글 정렬, 위지윅 편집은 이번 범위 밖이다. *(2026-07-21 갱신: 사이트 내 검색은 [002-redesign](../002-redesign/spec.md)으로 편입됨)*
- AI 에이전트의 자동 글감 캡처는 별도 프로젝트다. 이번 범위는 외부 도구가 초안을 넣을 수 있는 규약(입구)까지만 포함한다.
- 대상 독자는 일반 웹 브라우저(데스크톱·모바일) 사용자이며 별도 앱은 없다.
- 발행 전 확인은 에디터의 실시간 미리보기로 충분하며, 별도의 비공개 프리뷰 주소는 만들지 않는다 (grilling 확정).
- 태그는 자유 입력이며 사전 정의된 태그 목록 관리는 없다 (grilling 확정).
- 도메인·블로그 이름은 배포 시점에 설정값으로 주입하며 스펙에서는 미정으로 둔다 (grilling 확정).
- 조회수는 정밀 분석 지표가 아니라 "감을 잡는" 지표다. 유니크 방문자 집계는 하지 않는다 (grilling 확정).
